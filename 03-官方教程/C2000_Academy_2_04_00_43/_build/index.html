C2000 Academy

The C2000 Academy is a great resource for developers to learn about C2000 real-time microcontrollers platform. The Academy delivers easy-to-use training modules that span a wide range of topics for all C2000 devices. Additionally, C2000 Academy provides hands-on lab exercises and a full solution for each lab exercise. C2000 Academy is applicable to all generation-3 devices- F2807x, F2837xS, F2837xD, F2838xS, F2838xD, F28004x, F28002x, F28003x, F280013x and F280015x

Overview#

1. Getting Started

Texas Instruments

Getting started with C2000 Device

The objective of this module is to get started with TI Code Composer Studio (CCS) development environment and run a basic example to toggle LEDs. Specifically, this module demonstrates how to import a project from a C2000ware, modify/compile the code, and run the generated-out file on the target EVM (LaunchPad or controlCARD).

Texas Instruments

Getting started using SysConfig

TI’s Code Composer Studio includes a GUI-based tool for system configuration (SysConfig). It can be used for PinMux configuration, peripheral setup, and the generation of C-code using C2000Ware DriverLib API. This module goes over how to create a simple SysConfig project to toggle an LED.

Texas Instruments

Overview of C28x Devices and Accelerators

This introductory module includes an overview of the basic architecture of the C2000 family of real-time control microcontrollers.

3. System Control

Texas Instruments

Boot-Mode & Resets

This module describes the boot process and the various types of resets available on C2000 devices.

Texas Instruments

Interrupts

An interrupt is a response by the processor/CPU to an event that needs attention. This module goes over how to setup interrupts and the kinds of interrupts there are on C2000 devices.

Texas Instruments

Timers

This module describes C28x CPU Timers and the watchdog timer which is a safety feature that will reset the device if something goes astray in the system.

Texas Instruments

GPIOs & Crossbars

This module describes General Purpose Input Output (GPIO) pins, as well as crossbars. Crossbars provide the ability to route different signals throughout the device.

Texas Instruments

Memory Controller

This module describes the C28x memory controller including the types of Random Access Memory (RAM) (Ex. local shared RAM, global shared RAM, message RAM).

Texas Instruments

External Memory Interface (EMIF)

When an application requires more memory than what is available on-chip, the EMIF controller can be used to connect to external memory devices. This module goes into details about EMIF such as the features, configuration, and performance.

Texas Instruments

Flash Memory

This module describes C28x flash memory and programming flash memory.

4. Analog Subsystem

Texas Instruments

Analog to Digital Converter (ADC)

C2000 devices supports multiple analog-to-digital converters (ADCs). This module describes the different components in the ADC and how to setup the ADC.

Texas Instruments

Comparator Subsystem (CMPSS)

Each Comparator Subsystem (CMPSS) has two comparators within it. This module describes how the comparators can be used and setup.

Texas Instruments

Digital to Analog Converter (DAC)

The buffered DAC can be used to provide a programmable reference output voltage. This modules walks though the DAC features and the DAC block diagram.

Texas Instruments

Programmable Gain Amplifier (PGA)

Programmable Gain Amplifier (PGA) amplifies small input signals to increase the dynamic range of the downstream ADC and CMPSS modules. This modules showcases the benefits of using PGAs and the different parts of the block diagram.

Texas Instruments

Analog Interconnect

The Analog Subsystem Interconnect permits a single pin to route a signal to multiple analog modules. Thus, enabling a very flexible pin usage, allowing for smaller device packages.

5. Control Peripherals

Texas Instruments

Enhanced Pulse Width Modulation (ePWM)

The C2000 high-performance control peripherals are an integral component for all digital control systems. This module will cover pulse width modulation (PWM).

Texas Instruments

Enhanced Capture Module (eCAP)

The C2000 enhanced capture module allows time-based logging of external signal transitions. It is used to accurately time external events by time stamping transitions on the capture input pin.

Texas Instruments

Enhanced Quadrature Encoder Pulse (eQEP)

The Enhanced Quadrature Encoder Pulse (eQEP) module is essential when integrating C2000 devices in any application that requires motor control. The eQEP module enables the C2000 device to interact with linear or rotary incremental encoders to determine necessary motor information from rotating machines.

Texas Instruments

Sigma-Delta Filter Module (SDFM)

The C2000 Sigma Delta Filter Module (SDFM) is a four-channel digital filter designed specifically for current measurement in motor control applications.

6. Communication Peripherals

Texas Instruments

Communications Overview

C2000 devices contain features that allow several methods of communication and data exchange between the MCU and other devices. Many of the most commonly used communications techniques are presented in this module.

Texas Instruments

Fast Serial Interface (FSI)

The FSI module is a highly reliable high-speed serial communication peripheral capable of operating at dual data rate, providing 100 Mbps transfer using a 50 MHz clock.

Texas Instruments

Serial Communication Interface (SCI)

The SCI is a two-wire asynchronous serial port (also known as a UART) that supports communications between the processor and other asynchronous peripherals that use the standard non-return-to-zero (NRZ) format.

Texas Instruments

Inter-Integrated Circuit (I2C)

The I2C provides an interface between devices that connect using an I2C-bus and are compliant with I2C-bus specification version 2.1.

Texas Instruments

Serial Peripheral Interface (SPI)

The SPI is a high-speed synchronous serial port that shifts a programmable length serial bit stream into and out of the device at a programmable bit-transfer rate.

Texas Instruments

Multichannel Buffered Serial Port (MCBSP)

The McBSP provides a high-speed direct interface to CODECs, analog interface chips (AICs), and other serially connected A/D and D/A devices.

Texas Instruments

Controller Area Network (CAN)

CAN is a serial communications protocol which efficiently supports distributed real-time control with a high level of reliability.

Texas Instruments

CAN Flexible Data Rate (CAN FD/MCAN)

CAN FD refers to the protocol while MCAN refers to the physical module on the device that implements the CAN FD protocol.

Texas Instruments

Local Interconnect Network (LIN)

The LIN standard is based on the SCI (UART) serial data link format. The communication concept is single-commander/multiple-responder with a message identification for multi-cast transmission between any network nodes.

Texas Instruments

Power Management Bus (PMBus)

The PMBus module provides an interface for communicating between the microcontroller and other devices that are compliant with the System Management Interface (SMI) specification.

Texas Instruments

Ethernet for Control Automation Technology (EtherCAT)

Ethernet for Control Automation Technology (EtherCAT®) is an Ethernet-based field bus system.

Texas Instruments

Universal Serial Bus (USB)

The USB controller complies with USB 2.0 Implementers Forum certification standards, and utilizes full-speed (12 Mbps) operation in device mode and full-/low-speed (12 Mbps/1.5 Mbps) operation in Host mode.

7. Security & Safety Peripherals

Texas Instruments

Dual Code Security Module (DCSM)

The purpose of code security is to prevent unauthorized access to on-chip secure memories such as Flash, RAM, OTP and ROM. In DCSM there are secure zones to which securable resources can be assigned.

Texas Instruments

Functional Safety (FuSa)

Functional Safety-Compliant C2000 products can help streamline and speed up the ISO 26262, IEC 61508, and IEC 60730 certification processes with their built-in diagnostics, documentation, software, and support.

Texas Instruments

Background CRC (BGCRC)

A cyclic redundancy check (CRC) is an error-detecting code that is commonly used in digital systems. Some C2000 devices feature the background CRC (BGCRC) module, which computes the CRC value in hardware.

8. Debug & Diagnostics

Texas Instruments

Embedded Real-time Analysis and Diagnostic (ERAD)

The ERAD module enhances the debug and system analysis capabilities of the device. This module facilitates sophisticated non-intrusive detection and monitoring of the CPU operation in real-time.

Texas Instruments

C2000 Debugging Tips & Tricks

This module describes various topics related to debugging issues on the C2000 family of devices.

Texas Instruments

Embedded Pulse Generator (EPG)

The Embedded Pattern Generator (EPG) module is a customizable pattern and clock generator that could serve many test and application scenarios that require a simple pattern generator or a periodic clock generator. The EPG module can also be used to capture an incoming serial stream of data.

9. Advanced Topics

Texas Instruments

Control Law Accelerator (CLA)

The CLA is an independent 32-bit floating-point math hardware accelerator which executes realtime control algorithms in parallel with the main C28x CPU, effectively doubling the computational performance.

Texas Instruments

Direct Memory Access (DMA)

The DMA module provides a hardware method of transferring data between peripherals and/or memory without intervention from the CPU, effectively freeing up the CPU for other functions.

Texas Instruments

Configurable Logic Block (CLB)

The Configurable Logic Block (CLB) is a peripheral exclusive to C2000 devices that allows users to incorporate custom logic without the need for an external logic device such as an FPGA or CLPD.

Texas Instruments

Interprocessor Communications (IPC)

IPC allows communication between two CPU subsystems (ex. CPU1 and CPU2). This module explains the use and operation of IPC.

Labs

Texas Instruments

ADC Lab

Part one consists of sampling a single ADC input channel at a prescribed sampling rate and storing the conversion result in a circular memory buffer. Part two is to integrate the digital-to-analog converter (DAC) into part 1.

Texas Instruments

EPWM/ECAP Lab

Generate a PWM waveform using one of the ePWM modules. The period and duty cycle of the PWM waveform will be measured by one of the eCAP modules.

Texas Instruments

EQEP Lab

This lab illustrates a common use-case of the eQEP module- connecting quadrature signals to the C2000 device and using the eQEP to calculate motor characteristics.

Texas Instruments

SCI Lab

Utilize the SCI (Serial Communication Interface) by sending and receiving data between a C2000 device and a computer. The computer will be used to change the frequency of the blinking LED and then the board will echo this value back to the computer.

Texas Instruments

CLA Lab

Use the Control Law Accelerator (CLA) to implement a FIR lowpass filter. We will apply a PWM waveform to the input of our lowpass filter, which will result in a sinusoidal waveform at the output. The PWM waveform will be generated using one of the ePWM modules and it will be sampled via the ADC at a given sample rate.

Texas Instruments

DMA Lab

Use the Direct Memory Access (DMA) controller to regularly service the ADC in a real-time data processing application. The ePWM module will be used to generate a PWM waveform to simulate a real-time data signal, and the ADC will be setup to sample the PWM waveform at a chosen sampling rate. Accordingly, the DMA will be used to store the ADC samples in a ping pong buffer, so groups of samples can be processed by the CPU in an interrupt service routine (ISR).

Texas Instruments

CLB Lab

Gain familiarity with the Configurable Logic Block (CLB) peripheral and its submodules through a guided lab example. This lab demonstrations the CLB’s ability to connect to other peripherals, perform logical operations, override peripheral outputs, and then generate interrupts to the CPU.

Texas Instruments

eCAP Implementation Using CLB Lab

This lab demonstrates the CLB’s ability to connect to a signal, accurately time events, and send data to the CPU. The CLB will measure a high signal, a low signal, and another high signal before sending the data to the CPU. An external wave generator or the ePWM module can be set up to supply a signal, if a wave generator is not available.

Texas Instruments

IPC Lab

Use basic IPC features to send data in both directions between CPU1 and CPU2. PWM2 will be configured to provide a 50 kHz SOC signal to ADC-A. An End-of-Conversion ISR on CPU1 will read each result and write it into a data register in the IPC. An IPC interrupt will then be triggered on CPU2 which fetches this data and stores it in a circular buffer. The same ISR grabs a data point from a sine table and loads it into a different IPC register for transmission to CPU1. This triggers an interrupt on CPU1 to fetch the sine data and write it into DAC-B.

Texas Instruments

System Integration Lab

This lab builds on all of the different topics covered thus far. As a system example, the epwm module is setup to output a 50kHz output with a duty cycle that will vary based on an ADC conversion result. A comparator will also be used to monitor the ADC conversion results and cause a one-shot trip to drive the EPWM output low in case the ADC conversion is above a certain threshold.
